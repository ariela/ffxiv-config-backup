# **FF14 設定同期・バックアップツール アーキテクチャ設計書**

## **0\. 前提条件の検証と致命的欠陥の指摘（Brutal Honesty）**

ユーザーの要件には論理的な矛盾とシステムの安全性を脅かす重大な欠陥が含まれている。開発前に以下の修正を必須とする。

1. **「設定ファイルはバックアップ先ディレクトリに保存」の矛盾（Chicken-and-Egg問題）**
    * **欠陥**: アプリケーションが起動した際、「バックアップ先はどこか」を知るための設定ファイル自体がNAS上にある場合、アプリはどこを参照して良いか分からず起動に失敗する。
    * **解決策**: 設定を「ローカル設定（OSのAppData等）」と「共有設定（NAS上）」に強制分離する。ローカルにはバックアップ先（NASパス）のみを保持させる。**ローカル設定が存在しない（初回起動時など）場合は、バックアップ先となる共有ディレクトリパスを指定・登録させる初期セットアップUIを強制表示する。**
2. **排他制御の欠如によるデータ破損リスク**
    * **欠陥**: FF14クライアント（ffxiv\_dx11.exe）が起動中にリストア（上書き）を実行すると、ゲーム終了時にクライアント上のメモリデータで上書きされ、最悪の場合ファイルが破損し初期化される。
    * **解決策**: バックアップ/リストア処理の前にOSのプロセス一覧を取得し、FF14が起動中の場合は処理を強制ブロックするロジックを要件に追加する。

## **1\. システム要件・アーキテクチャ概要**

| コンポーネント | 技術スタック・仕様 | 役割 |
| :---- | :---- | :---- |
| **Backend** | Golang (Wails Core) | OSファイルシステム操作、プロセス監視、JSONパース |
| **Frontend** | Wails (React/Vue/Svelte \+ Tailwind CSS) | UI描画、ユーザー入力ハンドリング |
| **Storage** | ローカルFS \+ 共有ディレクトリ(SMB/NFS) | 実データ及びバックアップデータの保存 |

## **2\. データ構造・ファイル設計 (MECE)**

### **2.1 ディレクトリ構成**

NAS（バックアップ先）のディレクトリ構造を以下のように標準化する。

\<Backup\_Directory\_Path\>/ (NAS上の共有フォルダ)  
├── shared\_config.json      \# 共有設定ファイル（キャラ紐付け、実データパス等）  
└── backups/  
├── 20260604\_120000\_UUID/ \# バックアップ単位（タイムスタンプ+一意ID）  
│    ├── meta.json        \# このバックアップのメタデータ（名称、メモ）  
│    ├── MACROSYS.dat     \# アカウント共通マクロ  
│    ├── FFXIV\_CHR001/    \# 対象キャラクターのバックアップデータ  
│    │    └── ACQ.DAT, COMMON.DAT, ...  
│    └── FFXIV\_CHR002/  
└── 20260605\_090000\_UUID/

### **2.2 設定ファイル (JSON Schema)**

#### **A. ローカル設定 (%APPDATA%/FF14SyncTool/local.json)**

{  
"backup\_target\_directory": "\\\\\\\\NAS\\\\Shared\\\\FF14\_Backups"  
}

#### **B. 共有設定 (\<Backup\_Directory\_Path\>/shared\_config.json)**

{  
"game\_data\_path": "C:\\\\Users\\\\Default\\\\Documents\\\\My Games\\\\FINAL FANTASY XIV \- A Realm Reborn",  
"character\_mappings": {  
"FFXIV\_CHR001": "Alice (Main)",  
"FFXIV\_CHR002": "Bob (Sub)"  
}  
}

#### **C. バックアップメタデータ (meta.json)**

{  
"backup\_id": "20260604\_120000\_UUID",  
"created\_at": "2026-06-04T12:00:00Z",  
"name": "パッチ7.0直前バックアップ",  
"memo": "HUDレイアウト大幅変更前。念のため保存。",  
"contains\_common\_macro": true,  
"characters": \["FFXIV\_CHR001", "FFXIV\_CHR002"\]  
}

## **3\. アプローチの選択肢（バックアップ・リストア方式）**

### **3.1 バックアップ方式の選択**

| オプション | 概要 | メリット | デメリット・リスク | 適用条件 |
| :---- | :---- | :---- | :---- | :---- |
| **A: スナップショット（フル）** | 毎回全ての対象ファイルをコピーし、1つのディレクトリに独立して保存する。 | リストアが極めて容易。特定の時点への復元が確実。ファイル破損時の影響が局所化される。 | 容量を消費する。 | **【採用】** FF14の設定ファイルは全キャラ合わせても数MB〜数十MB程度であり、昨今のNAS容量を考慮すると容量問題は無視できるため採用。 |
| **B: 差分バックアップ** | 初回のみフル、以降は変更があったDATファイルのみを保存。 | 容量を極限まで節約可能。 | リストア時に過去の差分を辿って再構築するロジックが必要。バグの温床になりやすい。 | 容量要件が極端に厳しい場合のみ。本件では不採用。 |

### **3.2 リストア時のUI/UX方式**

| オプション | 概要 | 適用シーン |
| :---- | :---- | :---- |
| **A: 粒度ベース選択** | 「HUDレイアウトのみ」「マクロのみ」といった人間が理解できる論理名で選択させる。 | ユーザーが「どのファイルが何の設定か」を知らなくても操作できる。UXが極めて高い。**【採用】** |
| **B: ファイル名直指定** | COMMON.DAT, MACRO.DAT 等の物理ファイル名をそのまま表示して選択させる。 | 実装コストが最小。対象ファイルが増えた際のメンテナンスが不要。 |

## **4\. Wails (Backend Go) API仕様定義**

Frontendから呼び出されるGoの関数群。

1. GetLocalConfig() string, error
    * ローカル設定（NASのパス）を返す。未設定時（初回起動時）は空文字を返し、フロントエンド側でパス指定UIをトリガーする。
2. SetBackupDirectory(path string) error
    * ローカル設定にNASパスを書き込み、接続確認および共有ディレクトリの初期化（shared\_config.json生成等）を行う。
3. GetSharedConfig() SharedConfig, error
    * NAS上の共有設定（キャラ紐付け等）を返す。
4. UpdateCharacterMapping(charID string, name string) error
    * 共有設定のキャラクター紐付けを更新する。
5. CreateBackup(name string, memo string) error
    * 実データディレクトリから対象ファイル（MACROSYS \+ 各CHRディレクトリ内の指定ファイル）をコピーし、バックアップセットを生成する。
6. GetBackupList(charID string) \[\]BackupMeta, error
    * 指定したcharIDが含まれるバックアップの一覧を降順で取得する。（charIDが空の場合は全体マクロ等のバックアップ一覧とする）
7. RestoreBackup(backupID string, charID string, files \[\]string, includeCommon bool) error
    * 指定されたファイルをNASからローカルへ上書きする。フロントエンドからは対象となる物理ファイル名（マッピング解決後）の配列を受け取る。実行前にFF14プロセスの起動チェックを行う。

## **5\. UI設計 (3ペイン方式)**

UIモックアップは同梱のHTMLファイルを参照のこと。左から右へ依存関係が流れる設計（Left: 状態 \-\> Middle: その状態に対する履歴 \-\> Right: その履歴に対する詳細操作）はMECEであり、ユーザーの認知負荷を最小化する。

* **初期セットアップ画面 (Initial Setup Modal)**
    * アプリケーション起動時、ローカル設定が存在しない場合に強制表示されるモーダルまたは専用画面。
    * バックアップ先となるNASのパスを入力させ、有効性検証（書き込みテスト）後にローカルへ保存させる。
* **左ペイン (Character Selection)**
    * 実データまたはNASに存在する FFXIV\_CHR\* をリスト表示。
    * カスタム名（紐付け）が設定されていればそれを優先表示。インラインで名称編集ボタンを配置。
    * アカウント共通（MACROSYS.dat）用の特別な行を設ける。
* **中ペイン (Backup History)**
    * 左ペインで選択された要素に関連するバックアップのリスト（日時、名称）。
    * 各行にホバー等でメモ（Memo）を表示。
* **右ペイン (Restore Target Selection)**
    * 中ペインで選択されたバックアップに含まれる復元対象を**論理名（例：「HUDレイアウト」「マクロ」「ギアセット」等）のリストで表示**する（チェックボックス付き）。
    * フロントエンド内部で論理名から物理ファイル名（COMMON.DAT, MACRO.DAT等）へのマッピングテーブルを保持し、ユーザーには物理ファイル名を直接意識させない。
    * 「すべて選択/解除」トグル。
    * 最下部に「リストア実行」のCTAボタン（FF14起動中はDisabled制御）。
